Get all Secret Stores

Filter How to use
type Filter secret store by type.

Valid values: AWS_ASM, AZURE_AKV, or GCP_GSM.
For example:
GET https://<sub domain>.secretshub.cyberark.cloud/api/secret-store?filter=type EQ AWS_ASM
data.accountId Filter only the AWS targets defined under the same AWS account ID.

For example:
GET https://<sub domain>.secretshub.cyberark.cloud/api/secret-store?filter=data.accountId EQ 123456789100
data.azureVaultUrl Filter only the Microsoft Azure targets defined for a specific Azure Key Vault.

For example:
GET https://<sub domain>.secretshub.cyberark.cloud/api/secret-store?filter=data.azureVaultUrl EQ https://myVault.vault.azure.net/
data.appClientId Filter only the Azure secret stores with the same Azure app registration ID.

For example:
GET https://<sub domain>.secretshub.cyberark.cloud/api/secret-stores?filter=data.appClientId EQ MyAzureAppID

Create Secret Store

In Secrets Hub, a secret store is a representation of the object that stores your secrets.

Secret store Description
Target The target for the secrets, either where the secrets are scanned or where the secrets are syncing to.

Supported secret stores:
- AWS_ASM (AWS Secrets Manager)
- AZURE_AKV (Azure Key Vault)
- GCP_GSM (Google Cloud Secret Manager)
Source The source of the secrets, meaning where the secrets are syncing from.

Supported source secret stores: PAM_PCLOUD, PAM_SELF_HOSTED

The base URL for all Secrets Hub REST APIs:

https://<sub domain>.secretshub.cyberark.cloud/
Authorizations:
bearerAuth
Request Body schema: application/json
required
accountAlias
required
string (Accountalias) [ 3 .. 63 ] characters [a-zA-Z0-9!"#$%&'()*+,-./:;<=>?@[\]^_``{|}~]

The alias of your AWS account

accountId
required
string (Accountid) = 12 characters [0-9]

The 12-digit account ID of the AWS account that has the AWS Secrets Manager where you store secrets

regionId
required
string (Regionid) [ 4 .. 22 ] characters [a-z0-9-]

The region ID where the AWS account is managed

roleName
required
string (Rolename) [ 1 .. 64 ] characters [a-zA-Z0-9_+=,.@-]

The AWS role used to allow Secrets Hub to manage secrets in your AWS Secrets Manager

Filter options

The following table lists the filter expressions that can be used in the filter query parameter.

Note:

  • All values are case-insensitive.
  • All conditions can be grouped together with the AND logical operator.
  • Filters need to be formatted as specified in Secret Hub query language.

storeName Valid operator on field: CONTAINS https://url/api?filter=storeName CONTAINS value
vendorType Valid operator on field: EQ https://url/api?filter=vendorType EQ AWS

originId Valid operator on field: CONTAINS https://url/api?filter=originId CONTAINS org-id
discoveredAt Valid operator on field: GE,LE https://url/api?filter=discoveredAt GE "2021-09-03T12:42:07.105000Z"

vendorData.region Valid operator on field: EQ https://url/api?filter=vendorData.region EQ us-east-1
vendorData.tags.key Valid operators on field: CONTAINS, NOTCONTAINS https://url/api?filter=vendorData.tags.key NOTCONTAINS 'some key'
vendorData.tags.value Valid operators on field: CONTAINS, NOTCONTAINS https://url/api?filter=vendorData.tags.value CONTAINS 'some key'
